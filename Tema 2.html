¿Qué es la Accesibilidad Web?


Se refiere a la capacidad de un sitio web de ser utilizada por la mayor cantidad de personas independientemente de sus discapacidades, aptitudes limitadas o difícil acceso de manera que no le impida navegar y comprender o interactuar con la web.

Un sitio web accessible no impide a las personas con discapacidades visuales, auditivas, motoras o cognitivas ser utilizado por ellas para lograr su objetivo, sin embargo no solo se limita a este público si no tambien a los usuarios de dispositivos móviles donde el contenido se ve representado en pantallas mas pequeñas.

La accesibilidad web como la conocemos es principalmente propuesta por el grupo internacional W3C quienes bajo la iniciativa accesibilidad web o WAI en inglés buscan crear un estándar y normas, como el WCAG 1.0 y su versión actualizada WCAG 2.0. El principal cambio está en las 14 pautas originales que fueron reducidas a tan solo 4 principios que se exploran más adelante.

Existe cierto rechazo al proceso de volver un sitio web accesible, sin embargo es mucho más beneficioso para las compañias apelar por un público mayor sin discriminar o excluir diferentes grupos. Además el concepto de accesibilidad web está altamente relacionado a la usabilidad web por lo tanto su respectiva aplicación contribuye con la creación de un sitio web de mayor calidad.


Importancia


Su principal importancia radica en posibilitar el uso de un sitio web a personas con discapacidades, adultos mayores o incluso usuarios con conexión limitada a internet, brindando opciones de accesibilidad y cumpliendo con las reglas internacionales. La web por derecho de las personas debe ser accessible para todos en todo momento y lugar, además la web cuenta con infinidad de herramientas para facilitar la accesibilidad a diferencia de otros medios.

Otro punto importante es la gran cantidad de dispositivos inteligentes además de las computadoras que existen hoy en día como por ejemplo: dispositivos móviles, pantallas inteligentes o relojes inteligentes, cada uno con diferentes tipos de pantalla y diferente uso. La accesibilidad web tiene el papel de garantizar el uso del sitio web para las personas independientemente del dispositivo desde el cual ingresan.

En la actualidad la navegación web tiene una importancia considerable a nivel económico, laboral, académico y social en la vida diaria. Garantizar el libre acceso a toda la informacion en la web de manera cómoda para todos contribuye al avance en busca de una sociedad con las mismas oportunidades.


Tener un sitio web accesible es fundamental en los últimos años, despues de todo no se busca causar frustración a los usuarios que ingresen si no ampliar el alcance del sitio a una población mayor, rompiendo las barreras de uso que aún se encuentren en muchísimas páginas web.

Principios

  1. Perceptibilidad

    Se busca la implementación de alternativas para el contenido que no es presentado de manera que los usuarios que lo necesiten puedan percibir esta información. Para lograr esto se debe:


    Todo contenido no textual como imagenes debe contener un texto alternativo que describa el elemento.

    Proveer alternativas para contenido multimedia como transcripciones o subtítulos.

    Crear el contenido de manera que se pueda presentar de varias maneras.

    Usar contenido con elementos distinguibles haciendo buen uso de color, contraste, tamaño de las letras ajustable

    La opción de transcripción es un ejemplo de este principio en contenido multimedia.

  2. Operable

    La interfaz, controles y navegación es operable todo el tiempo. Para aplicar este principio se recomienda:

    Implementar sus funciones para teclado

    Brindar suficiente tiempo al usuario para leer el contenido, los límites de tiempo pueden ser detenidos o extendidos por el usuario.

    No crear contenido que pueda causar convulsiones como imágenes intermitentes. También se debe advertir ante cualquier contenido parpadeante.

    Ayudar al usuario a navegar entre el sitio y encontrar el contenido relevante. Utilizar titulos claros y evitar informacíon repetida.

    Implementar usos alternativos al teclado como botones táctiles, gestos o comandos por voz.

    Por ejemplo el carrusel en la página web de la UCR permite detener el desplazamiento automático.

  3. Comprensible

    Todo el contenido y acciones de la interfaz debe ser comprensible, se procura aplicar lo siguiente:

    El texto es legible utilizando un lenguaje simplificado e identificar los idiomas.

    Predecible, reptiendo la navegación por el sitio y elementos que realizan las mismas acciones.

    Ayudar a los usuarios a evitar, diagnosticar y corregir errores. Con opciones para deshacer acciones y mensajes de error entendibles.

    Ayudar al usuario a navegadar entre el sitio y encontrar el contenido relevante. Utilizar títulos claros y evitar informacíon repetida.

    Un ejemplo de sitio web comprensible es la página de inicio de la UCR cuyas secciones delimitan el contenido relevante con títulos claros.

  4. Robustez

    Asegurarse de que el contenido pueda ser interpretado por herramientas de usuario. Es importante realizar las siguientes comprobaciones:

    Asegurarse de que el maquetado no tenga errores de sintaxis.

    Establecer un nombre, rol y valor para todo componente fuera del estándar.

    Cómo ejemplo se muestra el resultado de un programa de comprobación tras examinar una URL.
