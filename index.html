¿Qué es la Usabilidad Web?


La usabilidad web es una propiedad que engloba un conjunto de características pensadas por el diseñador con el fin de garantizar la claridad, facilidad de uso y sobre todo la comodidad del usuario dentro de la página web, combina soluciones técnicas con aspectos visuales para llegar al mejor resultado. La facilidad de uso se mide por medio de las experiencias de los usuarios dentro de la página, su rápidez de navegación y satisfacción con los resultados

El mejorar este aspecto permite a los diseñadores web crear páginas de calidad en su manejo o tambien llamadas "User Friendly". Ser visualmente atractiva, sencilla de utilizar y buena estructura son algunas de las características que aumentan la usabilidad web de la página.

Para la Usabilidad Web Jakob Nielsen es uno de los pioneros más reconocidos en este campo y fue quien la definió como se le conoce hoy en día.

Propuso los siguientes 10 principios heurísticos para la usabilidad web que aún a día de hoy son usados por los diseñadores por sus excelentes resultados.


  • Visibilidad del estado del sistema

  • Relación entre el sistema y el mundo real

  • Control y libertad del usuario

  • Consistencia y estándares

  • Prevención de errores

  • Reconocer antes que recordar

  • Flexibilidad y eficiencia de uso

  • Diseño estético y minimalista

  • Ayudar a los usuarios a reconocer, diagnosticar y corregir los errores

  • Ayuda y documentación

Importancia


Su importancia radica en mejorar la experiencia final de los usuarios dentro de la página. Al ingresar al sitio el usuario busca cumplir un objetivo que como obtener informacion, comprar algún producto, contratar servicios, etc... La usabilidad web tiene el papel de facilitar al usuario el camino a cumplir sus objetivos, evitando la frustración y logrando la satisfacción.

El tener una página con la usabilidad web bien pensada también puede ayudar a aumentar el tráfico constante de usuarios, ya que si causa una buena impresión pueden volver a entrar en otro momento o mantener a los usuarios más tiempo dentro del sitio, ambos factores favoreciendo el posicionamiento SEO. En cambio si la usabilidad es altamente deficiente seguramente el usario final terminará abandonando la página web y penalizando el SEO del sitio web.


A continuación se profundiza en los principios propuestos por Jakob Nielsen.

Principios

  1. 1. Visibilidad del estado del sistema

    Indica que el usuario debe estar al tanto de lo que sucede en la página web para mantenerlo en control.

    En otras palabras brindar una pequeña retroalimentación tras cada acción u cambio para que el usurio sepa que sus acciones están dando resultados.

    Algunos ejemplos pueden ser el indicar la ubicación del usuario en la página o las famosas barras de progreso.

    Otro ejemplo de este principio son los Breadscrumbs ubicados arriba en la página y nos indican nuestra ubicación. Así también el usuario puede saber como llegó a la sección actual.

  2. 2. Relación entre el sistema y el mundo real

    No debemos esperar conocimiento especializado por parte de nuestros usuarios. Dentro de la página se debe manejar un lenguaje común y familiar, fácilmente comprensible por los lectores.

    Optar por usar íconos ya estandarizados para las diferentes opciones y estructurar de manera lógica el orden de la información es fundamental. De esta manera el usuario puede navegar naturalmente por un entorno mas familiar pues tendrá una noción de lo que está pasando.

    Por ejemplo la pagina de Librería Internacional utiliza el icono de una bolsa para las compras.

  3. 3. Control y libertad del usuario

    Las personas a menudo cometen errores y en una página web no es excepción. El no poder corregir estos errores puede provocar frustración en los usuarios, para ello se deben colocar opciones para deshacer la acción realizada de la manera mas sencilla posible. Con esto evitando el castigo al usuario.

    Para ciertas situaciones es hasta recomendable enviar una notificación de confirmación antes de efectuar cualquier cambio.

    Un ejemplo apropiado es la página de imágenes Pinterest que si por error ocultas un "pin" puede deshacer la opción.

  4. 4. Consistencia y estándares

    La página web debe aplicar convenciones ya establecidas en lo posible, facilitando el uso del usuario al no tener que aprender nuevas instrucciones para realizar acciones ya conocidas, pero sobretodo, que el funcionamiento de la página sea coherente por, ejemplo manteniendo el nombre de sus elementos en toda la página.

    Un claro ejemplo de este principio es la utilización del ya conocido ícono de lupa para las búsquedas.

  5. 5. Prevención de errores

    Antes de pensar una página que permite la correción de errores se debe pensar en una página que evite que sucedan estos posibles errores.

    Para estose pueden implementar sistemas que briden ayuda al usuario a navegar u seleccionar opciones filtrando resultados.

    La página de alojamientos Booking muestra opciones únicamente si están diponibles en ese periodo.

  6. 6. Reconocer antes que recordar

    Busca que el usuario no tenga que esforzarse por recordar información, si no brindar ayuda para que el usuario pueda reconocer facilmente, memorizar puede ser abrumador en un inicio pero los usuarios suelen reconocer patrones si están bien elaborados.

    Se puede logar con íconos facilmente reconocibles o con una vista previa a los resultados de una acción.

    Por ejemplo la página de streaming Twitch posee diferentes categorías para que el usuario vea directos según temas predefinidos sin la necesidad de buscar.

  7. 7. Flexibilidad y eficiencia de uso

    Indica la posibilidad de que usuarios principiantes puedan usar la página sin problema alguno y los usuarios expertos puedan profundizar en sus funciones avanzadas para mayor eficiencia, creando así una experiencia personalizada a cada tipo de usuario.

    Los atajos u opciones de personalización fomentan el interés de los usuarios por aprender a manejar estos sistemas que brinda la página.

    Un ejemplo conocido es la página de diseño Canva que ofrece muchisimas opciones ya listas para usuarios principiantes pero esconde funciones avanzadas para usuarios más conocedores, como las opciones de paletas de colores.

  8. 8. Diseño estético y minimalista

    Es importante no sobrecargar al usuario con demasiada informacion si no mostrar lo justo y necesario.

    Permite diseños más visibles y atractivos lo que ayuda a la orientación clara del usuario. Una página Por más opciones que tenga puede diseñarse para lograr una vista sencilla y efectiva sin abrumar al usuario final.

    Hay muchos factores estéticos que influyen en este punto como los colores elegidos, íconos, tipografía e imágenes.

    La página off white posee un diseño bastante sencillo de entender con solamenta la información necesaria en pantalla.

  9. 9. Ayudar a los usuarios a reconocer, diagnosticar y corregir los errores

    Es probable que en algún momento se produzca un error que los desarrolladores no previeron. En estos casos se opta por enviar un mensaje de error comprensible para el usuario que le de una idea de lo sucedido, las posibles causas y recomendaciones a seguir para corregir el error.

    Por ejemplo los inicios de sesión incorrectos a menudo muestran un mensaje por campos incompletos u contraseñas de formato no establecido.

  10. 10. Ayuda y documentación

    Es de vital importancia tener en una página una sección de ayuda que actue como guía para el usuario en caso de ser necesario. Puede contener informacion relevante, preguntas frecuentes e incluso contactos en caso de cualquier inconveniente que lo amerite.

    Pinterest posee un apartado del estilo, de facil acceso y claridad.
